Beginner path

Write programs as grammatical Sanskrit.

Sanskript is small today, but its shape is already unusual: case endings and verbal frames decide what the program means. Start here, write real examples, and watch the grammar become executable.

First Program

Source

gaṇakaḥ pañca phale nidadhāti.
gaṇakaḥ phalaṃ dvābhyāṃ vardhayati.
gaṇakaḥ phalaṃ darśayati.

Meaning

The reckoner places five in the result, increases the result by two, and displays it.

Output: 7

The Four Moves

Place

nidadhāti stores a value in a locative location.

Increase

vardhayati increases an accusative target by an instrumental amount.

Decrease

nyūnayati decreases an accusative target by an instrumental amount.

Display

darśayati prints the accusative value.

Cases Drive Meaning

Case Sanskrit role Programming use Example
Nominative kartṛ agent gaṇakaḥ
Accusative karman value or target phalaṃ
Instrumental karaṇa tool or amount dvābhyāṃ
Locative adhikaraṇa storage location phale

Write And Run

Create

Save a source file with the extension .ssk.

Run

Use the Python module runner from the repo root.

$env:PYTHONPATH='src'; python -m sanskript examples\prathama.ssk

Explore Forms

Current grammar infrastructure includes subanta declension, pronouns, kāraka mapping, IAST/Devanagari transliteration, and first-pass sandhi.

Current Boundaries

Honest compiler contract

Sanskript is expanding under a stricter canon gate. Adhyāya 1 through 6 are partial until each sutra has discrete executable logic with exact source text, inherited domains, conditions, exceptions, positive behavioral tests, and negative behavioral tests. A canon item marked batch_partial has subsystem support, but it is not yet complete sutra-by-sutra. Accepted source is still a controlled subset, because beautiful grammar beats fake breadth.
